1. JPA - Hibernate

  • 다양한 연관 관계를 효과적으로 관리하여 비즈니스 도메인 요구사항 구현

다양한 연관 관계를 활용하여 엔터티 간의 복잡한 데이터 모델을 설계하고, 비즈니스 도메인에 필요한 기능을 효율적으로 구현할 수 있습니다. 이를 통해 프로젝트 요구사항에 따라 다양한 도메인의 애플리케이션을 개발할 준비가 되어 있습니다.

  • 비기능 요구사항까지 고려한 연산 전략과 엔터티 관리 능력

다양한 Fetch 전략 (EAGER, LAZY)을 이해하고, 데이터베이스 조회의 효율성을 고려하여 올바른 연산 전략을 선택할 수 있습니다. 엔터티 매니저를 사용하여 영속성 컨텍스트를 관리하고, 엔터티의 상태 변화를 감지하여 데이터베이스와의 동기화를 수행할 수 있습니다.

  • 데이터베이스 모델링 및 쿼리 최적화

OneToMany, ManyToOne 및 ManyToMany 관계에 대한 심층적인 이해를 바탕으로 특정 비즈니스 도메인 요구 사항을 충족하기 위한 올바른 어노테이션 사용 및 엔터티 구성을 할 수 있습니다.
유연한 쿼리 작성 및 애플리케이션 성능 향상을 위해 사용자 정의 타입 매핑 및 특정 데이터 검색을 위한 프로젝션 생성자를 구축할 수 있습니다.

2. Functional Programming

  • Java Stream API를 통한 데이터 정밀 처리

컬렉션 데이터를 함수형으로 처리할 수 있는 스트림 API를 활용하여 데이터 변환, 필터링, 정렬 등을 수행할 수 있습니다. 이를 통해 데이터 처리 작업을 더욱 정밀하게 수행하고, 효율적인 코드를 작성할 수 있습니다.

  • 효율적이고 재사용 가능한 코드를 위한 함수형 인터페이스 활용

Java 8 함수형 인터페이스인 Function 및 Consumer를 활용하여 효율적이고 재사용 가능한 코드 구조를 설계할 수 있습니다. 함수형 인터페이스를 활용한 유틸리티 클래스를 작성하여 반복되는 데이터베이스 작업을 캡슐화하고, 가독성과 유지보수성이 높은 코드를 작성할 수 있습니다.

  • 프레임워크와 함수형 프로그래밍 통합 능력

Spring의 AbstractHttpConfigurer 객체와 같은 다양한 컨텍스트에서 함수형 프로그래밍을 적용할 수 있습니다. 또한, MyBatis 메소드와 Stream API를 통합하여 주 키(primary key) 참조에 기반한 효율적인 연관 관계 조인을 수행함으로써 영속성 계층과의 상호작용을 최적화할 수 있습니다.

요청 URL의 파라미터를 활용하여 클라이언트 및 애플리케이션 상 코드의 재사용성을 높이고, Bootstrap의 Spinner 및 Toast를 이용하여 비동기 프로그래밍의 응답성을 유지할 수 있습니다. Promise Chaining을 통해 비동기 프로그래밍 코드를 간결하게 유지하고, 비즈니스 구현을 위한 함수의 실행 과정을 명확하게 작성할 수 있습니다.

4. Spring Framework

  • Spring MVC
    • 리소스 매핑: URL을 적절한 뷰 및 템플릿에 매핑하여 Spring MVC의 뷰 템플릿 관리 능력을 적절하게 운용할 수 있습니다.

    • 사용자 입력 처리: @RequestParam, @PathVariable, @RequestBody 및 사용자 정의 커맨드 객체와 같은 다양한 방법을 통해 사용자 입력을 효과적으로 가져옴으로써 클라이언트로부터 데이터를 수집하는 과정을 최적화할 수 있습니다.

    • RESTful 디자인: 클라이언트로부터 API 엔드포인트에 대한 요청을 보내는 방법을 이해하고 있습니다. 비동기적 웹 개발 기술을 활용하여 요청 및 응답을 처리하고, 클라이언트 측의 상호작용을 개선하는데 능숙합니다.

  • Spring Data JPA
    • Spring Data JPA 네이밍 규칙 이해 및 활용: Spring Data JPA의 네이밍 규칙을 숙지하고 있으며, 자동으로 생성되는 쿼리 메서드를 활용하여 데이터베이스 연동을 간편하게 처리할 수 있습니다.

    • JPQL을 활용한 데이터베이스 쿼리 정의: JPQL (Java Persistence Query Language)을 사용하여 복잡한 데이터베이스 쿼리를 정의하고 실행한 경험이 있습니다. 이를 통해 Spring Data JPA 네이밍 규칙에 의해 제공되는 연산보다 더 복잡한 데이터 추출 및 조작 연산을 구현할 수 있습니다.

    • 효율적인 데이터 연산: 프로젝션, 페이징 및 정렬 기능을 Spring Data JPA를 통해 구현함으로써 대량 데이터를 효율적으로 관리하고 클라이언트에게 최적화된 결과를 제공할 수 있습니다.

  • Spring Security
    • 확장 가능한 Security 로직 구현: UserDetails에 Decorator 디자인 패턴을 적용하여 보안 관련 세부 정보를 캡슐화함으로써 도메인 모델과 Spring Security의 보안 객체 간의 기능을 분리하고 둘 간의 결합도를 낮추어 확장 가능한 코드베이스를 작성할 수 있습니다.

    • 대화형 Security 디자인: RESTful API 요청과 Spring Security 간 통합을 통해 로그인 폼 기반의 웹 인증을 구현하는 방법을 숙지하고 있습니다.

    • Security 플로우 상세 구현: 커스텀 Provider와 Filter를 구현하여 사용자의 인증 및 인가 과정을 조정하고 확장할 수 있는 능력을 갖추고 있습니다.
